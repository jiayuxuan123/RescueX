# RescueX v3.4.1

> 正式发布构建。

## 本次更新

### 全新工作台 UI（第二版）
- 重构为移动优先的中性白/灰设备工具界面：更少视觉噪声、更清晰的状态层级、更大的触控目标。
- 页面改为四个明确功能入口：概览、保护设置、模块恢复、工具与日志；一次仅展示当前工作所需的内容。
- 首页只显示当前安全状态、启动摘要与统计，并提供三个写明用途的操作：刷新状态、运行完整性检查、保护设置。
- 已移除聊天式输入栏、发送箭头、固定底部栏和侧栏跳转；这些模式不适用于救砖工具。
- 所有高风险操作仍在原位置，沿用原有确认逻辑；没有将危险操作放入快捷主按钮。

### 交互与可靠性
- 手动刷新增加并发保护，避免快速重复点击触发重叠刷新。
- 工作流导航直接绑定实际功能卡片，不依赖固定页面位置。
- 保留全部既有 `data-action`、表单 ID、状态容器和 KernelSU/Magisk 桥接协议。
- 非 KernelSU/APatch/Magisk WebUI 环境仍会拒绝运行，未加入生产调试后门。

## 建议测试
1. 在 KernelSU / APatch / Magisk v27+ WebUI 中安装 ZIP 并打开 WebUI。
2. 检查中英文切换、侧栏所有入口、底部“刷新 / 报告 / 检查”按钮。
3. 验证配置保存、模块白名单、快照、日志切换、报告导出。
4. 确认危险操作仍会出现明确确认，且不会误触。
5. 检查窄屏、深色系统主题和横竖屏下是否有遮挡或横向滚动。
